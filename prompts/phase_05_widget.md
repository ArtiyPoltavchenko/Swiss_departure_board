- `android/app/src/main/res/drawable/` — widget background drawable

## Context
Read `CLAUDE.md` before starting.
Phase 4 (full UI) is complete. The app has real design, settings, localization, disruption badges.
Now add an Android Home Screen Widget that shows 3-4 departures from the nearest stop.

**⚠️ CRITICAL**: The widget runs as a native Android RemoteViews. It does NOT render Flutter widgets. The layout is pure XML. Data is passed from Dart via the `home_widget` package which writes to SharedPreferences and triggers native widget updates.

## Requirements

### 1. Widget Layout (`widget_layout.xml`)

Design: compact dark card matching app aesthetic.

```
┌─────────────────────────────────┐
│ 🚏 Bellevue                  🔄│  ← stop name + refresh button
├─────────────────────────────────┤
│ 7   Wollishofen          4 min │
│ 31  Hegibachplatz         7 min │
│ S6  Uetikon              12 min │
│ 33  Tiefenbrunnen         15 min│
└─────────────────────────────────┘
```

Implementation:
- Root: `LinearLayout` vertical, dark background (#1a1a2e), rounded corners via drawable
- Header row: `RelativeLayout` — stop name (TextView, white, bold), refresh ImageView (right-aligned, `ic_refresh` or similar)
- Departure rows: 3-4 `LinearLayout` horizontal rows, each with:
  - Line number (TextView, fixed width, colored background)
  - Destination (TextView, weight=1, ellipsize end)
  - Countdown (TextView, fixed width, bold, right-aligned)
- Use `@android:color/` or define custom colors in `res/values/colors.xml`

### 2. Widget Metadata (`widget_info.xml`)

```xml
<?xml version="1.0" encoding="utf-8"?>
<appwidget-provider xmlns:android="http://schemas.android.com/apk/res/android"
    android:minWidth="250dp"
    android:minHeight="110dp"
    android:resizeMode="horizontal|vertical"
    android:updatePeriodMillis="600000"
    android:initialLayout="@layout/widget_layout"
    android:widgetCategory="home_screen" />
```
Update period: 10 minutes minimum (Android enforces this). Actual frequent updates via WorkManager.

### 3. Widget Data Flow (Dart side)

`lib/services/widget_service.dart`:
```dart
class WidgetService {
  /// Fetch departures and write to SharedPreferences for widget.
  /// Called from WorkManager callback AND from app foreground.
  static Future<void> updateWidgetData() async {
    // 1. Read last known stop from SharedPreferences
    //    (saved by app whenever user selects a stop)
    // 2. If no saved stop: try geolocation (may fail in background)
    // 3. Fetch departures (top 4) via TransportApi
    // 4. Write to SharedPreferences keys:
    //    - "widget_stop_name": String
    //    - "widget_departure_0_line": String
    //    - "widget_departure_0_dest": String  
    //    - "widget_departure_0_time": String ("4 min" or "Now")
    //    - ... repeat for 1, 2, 3
    //    - "widget_departure_count": int
    // 5. Call HomeWidget.updateWidget() to trigger native refresh
  }
}
```

### 4. WorkManager Background Task

In `lib/main.dart`, register WorkManager:
```dart
void callbackDispatcher() {
  Workmanager().executeTask((task, inputData) async {
    await WidgetService.updateWidgetData();
    return true;
  });
}
```

Schedule periodic task:
```dart
Workmanager().registerPeriodicTask(
  'widget-refresh',
  'widgetRefresh',
  frequency: Duration(minutes: 15),
  constraints: Constraints(networkType: NetworkType.connected),
);
```

### 5. Refresh Button

The 🔄 button in widget header:
- On click: trigger `HomeWidget.widgetClicked` → Dart callback → `WidgetService.updateWidgetData()`
- While updating, button could show a different state (optional, RemoteViews animations are limited)

### 6. Widget → App Launch

Tapping anywhere on the widget (except refresh button) opens the main app.
Use `HomeWidget.widgetClicked` or set PendingIntent on the root layout.

### 7. Native Android Code

The `home_widget` package handles most of the bridge, but you may need to:
- Create/update `HomeWidgetProvider.kt` (or use the one generated by home_widget)
- In `onUpdate`, read SharedPreferences and populate RemoteViews
- Map the "widget_departure_N_*" keys to the TextViews in layout

Follow home_widget package documentation for the Kotlin/Java side.
